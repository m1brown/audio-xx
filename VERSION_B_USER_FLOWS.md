Entry flows:
1. New user → music input → headphones/speakers → system building guidance
2. Existing system → "evaluate my system" → system assessment with strengths/weaknesses/direction
3. Shopping → "I want to buy a DAC" → category + budget → recommendations with system context
4. Diagnosis → "my system sounds harsh" → causal reasoning → targeted direction
5. Comparison → "Chord vs Denafrips" → side-by-side trade-off framing
6. Hypothetical → "what if I replaced my DAC with an R2R?" → counterfactual reasoning

Decision support:
- Every recommendation includes "do nothing" as an option
- Every direction includes trade-off framing
- Confidence level is visible to the user
- System context shapes every response
