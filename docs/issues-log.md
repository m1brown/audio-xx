# Issues Log

Append-only log of bugs, behavior issues, and friction points found during testing or use. Either AI assistant (Claude or ChatGPT) can read this file when triaging or fixing.

## Format

```
## YYYY-MM-DD — Short title
**Source:** [you / friend name / testing harness / monitoring]
**Severity:** [low / medium / high / blocker]
**Repro:** how to reproduce
**Expected:** what should happen
**Actual:** what does happen
**Tags:** advisory-quality | ui | routing | data | perf | a11y | other
**Status:** open / investigating / fixed in <commit-or-PR> / wontfix
```

## How to use

- New issues append at the **top** under the next date heading
- Resolved issues stay in the file (don't delete) — the log is also a regression catalog
- For bugs you can fix in <5 minutes during a session, fix and log together
- For larger issues, log first, then schedule
- When passing to an AI assistant, paste a single open issue + relevant file paths — not the whole log
